# TALES Deployment Guide for Render

This guide will help you deploy TALES to Render.com.

## Prerequisites

- GitHub account with the tales_project repository
- Render.com account (free tier available)
- Google OAuth credentials configured
- API keys for Gemini and Perplexity

## Deployment Steps

### 1. Connect Your Repository to Render

1. Go to https://render.com and sign in
2. Click "New +" and select "Blueprint"
3. Connect your GitHub account if not already connected
4. Select the `tales_project` repository
5. Render will automatically detect the `render.yaml` file

### 2. Configure Environment Variables

During the setup, you'll need to provide these environment variables:

#### Backend Environment Variables

**Required:**
- `SECRET_KEY` - Will be auto-generated by Render
- `ENCRYPTION_KEY` - Will be auto-generated by Render
- `GOOGLE_CLIENT_ID` - Your Google OAuth Client ID
- `GOOGLE_CLIENT_SECRET` - Your Google OAuth Client Secret
- `GEMINI_API_KEY` - Your Google Gemini API key
- `PERPLEXITY_API_KEY` - Your Perplexity API key

**Optional:**
- `DATABASE_URL` - Already set to `sqlite:///./data/tales.db`

#### Frontend Environment Variables

**Required:**
- `VITE_API_URL` - Will be set to your backend URL (e.g., `https://tales-backend.onrender.com`)
- `VITE_GOOGLE_CLIENT_ID` - Same as backend Google Client ID

### 3. Deploy

1. Click "Apply" to create the services
2. Render will:
   - Build the Docker image for the backend
   - Build the static site for the frontend
   - Deploy both services

3. Wait for the deployment to complete (typically 5-10 minutes)

### 4. Post-Deployment Configuration

#### Update Google OAuth Settings

After deployment, update your Google Cloud Console OAuth settings:

1. Go to https://console.cloud.google.com
2. Navigate to your TALES project
3. Go to "Credentials" → "OAuth 2.0 Client IDs"
4. Add your Render URLs to:
   - **Authorized JavaScript origins:**
     - `https://tales-frontend.onrender.com` (or your custom domain)
   - **Authorized redirect URIs:**
     - `https://tales-frontend.onrender.com`
     - `https://tales-frontend.onrender.com/`

#### Update Frontend Environment Variable

1. In Render, go to your frontend service
2. Update `VITE_API_URL` to your actual backend URL
3. Trigger a manual deploy to rebuild with the correct URL

## Service Architecture

### Backend Service
- **Type:** Web Service (Docker)
- **Port:** 8000
- **Database:** SQLite with persistent disk (1GB)
- **Auto-scaling:** Yes (on paid plans)

### Frontend Service
- **Type:** Static Site
- **Framework:** Vite + React
- **CDN:** Enabled by default
- **HTTPS:** Enabled by default

## Database Persistence

The backend uses a persistent disk mounted at `/app/data` to store the SQLite database. This ensures your data persists across deployments.

## Troubleshooting

### Build Failures

If the backend fails to build:
- Check that all dependencies in `requirements.txt` are valid
- Review build logs in Render dashboard

If the frontend fails to build:
- Ensure `package.json` and `package-lock.json` are committed
- Check for Node.js version compatibility

### Runtime Errors

**Backend not connecting to database:**
- Check that the disk is properly mounted
- Verify `DATABASE_URL` environment variable

**Frontend can't reach backend:**
- Verify `VITE_API_URL` is correct
- Check CORS settings in backend (`app/main.py`)

**Google OAuth not working:**
- Verify all OAuth redirect URIs are configured
- Check that Client ID matches in both frontend and backend

## Scaling

### Free Tier Limitations
- Backend spins down after 15 minutes of inactivity
- First request after spin-down will be slower (cold start)
- 750 hours/month of runtime

### Upgrade Options
- **Starter Plan:** Always-on instances, no cold starts
- **Standard Plan:** More resources and auto-scaling
- **Pro Plan:** Enhanced performance and priority support

## Custom Domain

To use a custom domain:

1. In Render, go to your frontend service
2. Click "Settings" → "Custom Domain"
3. Add your domain name
4. Update DNS records as instructed
5. Update Google OAuth settings with new domain

## Backup Strategy

### Database Backups

The SQLite database is on a persistent disk. To backup:

1. Use Render's disk snapshot feature (paid plans)
2. Or, add a backup endpoint to download the database
3. Schedule regular backups to external storage (S3, etc.)
